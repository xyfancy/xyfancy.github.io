创建个人知识库

Veröffentlicht am 2017-11-18

经过这几天对于Github Wiki的学习和使用，越发觉得个人知识库的重要性。在人工智能如火如荼的今天，知识库的重要性已经不言而喻了。从原来古人在岩壁上记录下信息，到在龟壳上，竹简上，纸上，到如今的各种各样的存储介质，数据中心等等。人的记忆力是有限，将学习到的东西记录下来，形成自己的知识库是很有必要的。

介绍

Wiki（维基），在我看来就是现代化[1]的一种知识记录的系统。著名的维基百科就是一个巨大的知识库。但是就跟书籍一样，你去到图书馆也不可能想看所有的书，也很难看完所有的书。对于我们每个人来说，想获取的知识不尽相同，所以有必要形成自己的个人知识库来收录自己感兴趣的知识，其实也就相当于学习笔记。

有的人喜欢通过用笔记录，对于我来说可能更倾向于使用计算机来记录，而且我认为计算机能更有效地完成记录工作。同样是为了数据存储，为何不直接使用数据库而是Wiki呢？其实Wiki你也可以当作一个广义上的数据库，当我们选择工具的时候经常会问自己什么样的工具好，其实这个是没有定论的（每个人看法不同），在我看来对于知识的记录不需要多么复杂的管理系统以达到你得写一些SQL语言或者脚本来进行管理的地步，单纯记录东西我感觉wiki就够用了，特别是GitHub提供了远程的免费托管，这对于我们知识库的稳定保存是有好处的。所以当我看到GitHub Wiki的时候感觉这个就是我想要的东西。

GitHub Wiki

下面简单说一下GitHub Wiki。

  • 开启GitHub Wiki服务

GitHub Wiki服务开启很简单，你只需要在指定的仓库的Settings中勾选Wikis就行了。

  • Wiki的构成

    在GitHub Wiki中用户编辑的主要有4个主要部分，分别是Home,_Footer,_Sidebar和用户自己创建的Page。其实Wiki的各个部分用来做什么都是根据个人的想法和需求。所以下面谈谈我各部分的使用。

    Home,即首页，用来描述该Wiki记录的内容。

_Footer，即页脚，放上个人的一些联系信息，如邮箱和一些博客地址。

_Sidebar，即侧边栏，可以用来形成各个页的索引，如果使用Markdown，就是生成对应页的隐性链接。

Page，用户自己创建的页，可以用来记录自己想记录的，推荐GitHub支持的Markdown进行记录。

  • 一点看法

由于GitHub本着开源开放的思想，很多Wiki其实都是可以直接Clone下来的，但是个人知识库的目的不在于单纯地记录，而且体会记录和掌握知识的过程。

如果只是单纯的建立各个仓库的Wiki其实对于后面的查找也是不方便的，但是如果将所有的知识记录在一个Wiki上也是很难管理的，所以我认为可以再建立一个Wiki提供其他各个Wiki的链接。所以，我在该博客的仓库下建立了一个Wiki：https://github.com/xyfancy/xyfancy.github.io/wiki

搭建个人博客

Veröffentlicht am 2017-10-22

介绍

这篇博文主要用来说明GitHub Pages和Hexo配合搭建自己的个人博客。

在搭建之前，首先得弄明白Git Pages 和 Hexo是什么东西。
所有的工具在使用它之前首先得明白它的功能和使用方法。

What is GitHub Pages

GitHub Pages is a static site hosting service.

上面对于GitHub Pages的定义来自于 https://help.github.com/articles/what-is-github-pages/,from 。我们可以看出GitHub Pages就是GitHub提供的一种服务，其功能就是作为静态网站托管。简单的说就是你可以将你的静态网站通过该服务进行托管。

GitHub Pages is designed to host your personal, organization, or project pages directly from a GitHub repository. To learn more about the different types of GitHub Pages sites, see “User, organization, and project pages.”

You can create and publish GitHub Pages online using the Jekyll Theme Chooser. If you prefer to work locally, you can use GitHub Desktop or the command line.

这里简单描述了GitHub Pages的一些应用场景和使用方式。如果深究GitHub Pages的原理是比较耗时的，但是所有的服务都是基于某种特定的协议的，至于这种协议或者说规则是什么，得具体了解它的原理，甚至得看些源码。所以，对于大多数搭建个人博客的用户来说，如果从0开始使用GitHub Pages搭建博客所花费的时间是不值得的。 好在Hexo能帮你解决这些问题。

what is hexo

Hexo is a fast, simple and powerful blog framework. You write posts in Markdown (or other languages) and Hexo generates static files with a beautiful theme in seconds.

从 https://hexo.io/docs/ 中可以看出，Hexo就是为了搭建博客所设计的一种框架，通过这种框架就能很快的实现博客的搭建了。

安装

为了搭建个人博客需要进行下面的安装 :

  • Git .
  • Node.js.
  • hexo

Hexo的安装可以使用下面的命令行:

$ npm install -g hexo-cli

具体的细节可查看官方文档 https://hexo.io/docs/.

搭建

先建立一个目标文件夹<floder>用以进行Hexo的初始化，然后在终端中敲入下面命令：

$ cd floder

$ hexo init

$ npm install

如果一切顺利，可以看到目标文件夹下生成了许多的文件，其中有一个名为
_config .yml 的文件，这个文件就是配置文件，可以修改文件中对应行的参数进行博客网站的一些设置的修改。具体可提供的配置选项可以参考https://hexo.io/docs/configuration.html 。

为了将Hexo部署到GitHub,并且使用GitHub Pages的服务，得遵循GitHub约定的一些规则。 就以个人账户的博客为例，需要创建一个名为”username.github.io”的仓库，然后以该仓库作为部署的目标仓库。(其他类型使用GitHub Pages的规则可参考https://help.github.com/articles/user-organization-and-project-pages/).

然后打开刚才提到的_config .yml 文件，修改部署部分的参数。

deploy:

type: git

repository: git@github.com:username/username.github.io.git

branch: master

如果上述步骤正确可以依次输入以下命令完成部署：

$ hexo g

$ hexo d

部署成功后，就可以在 https://username.github.io/ 中查看你部署的网站了.
如果你出现了以下错误:

ERROR Deployer not found: git

可能需要输入下面的命令进行解决：

$ npm install hexo-deployer-git –save

发布和管理

写博客和管理博客的方法均可在 https://hexo.io/docs/ 中查看.
