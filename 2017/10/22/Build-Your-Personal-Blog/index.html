Build Your Personal Blog

Introduction

This blog is writtern to show my understanding of GitHub Pages adn Hexo.And I hope this blog can help you know how to build your own blog with GitHub Page and Hexo.

I always begin with the function of the tools or services I used so that I can figure out how much I should know about them.

What is GitHub Pages

“GitHub Pages is a static site hosting service.”

The describtion above is copied form the https://help.github.com/articles/what-is-github-pages/,from which, we can know GitHub Pages is a kind of service provied by GitHub. The function is obvious–Static Site Hosting.

GitHub Pages is designed to host your personal, organization, or project pages directly from a GitHub repository. To learn more about the different types of GitHub Pages sites, see “User, organization, and project pages.”

You can create and publish GitHub Pages online using the Jekyll Theme Chooser. If you prefer to work locally, you can use GitHub Desktop or the command line.

So you can hardly figure out the principle of this service for that the source codes must be read. Is it worthy to read a lots of source codes if you can get them？ I think if you just want to learn how to build a site to write and manage your blog. The best way is just use the service. So even though you can learn the framwork of Jekyll and can write something differnt,I consider the contents of blog is what matters.
That is why I choose Hexo.

what is hexo

Hexo is a fast, simple and powerful blog framework. You write posts in Markdown (or other languages) and Hexo generates static files with a beautiful theme in seconds.

From https://hexo.io/docs/ we know use hexo can save a lot of time to learn about Jekyll.

Installion

What you need are :

  • Git .
  • Node.js.
  • hexo

hexo can be installed with the command:

$ npm install -g hexo-cli

More details can get from https://hexo.io/docs/.

Set Up

Make a target <floder>,and you can initialise Hexo in it.

$ cd floder

$ hexo init

$ npm install

You can see some files generated in the target floder.
_config .yml is configuration file.https://hexo.io/docs/configuration.html can help you figure out how to set the property.

For delopying the hexo to github, you need open the _config.yml and you must create a repository with the name “username.github.io”(https://help.github.com/articles/user-organization-and-project-pages/).

deploy:

type: git

repository: git@github.com:username/username.github.io.git

branch: master

Then you can run the command below:

$ hexo g

$ hexo d

If succeed, Your site is published at https://username.github.io/.
And normally, you may need

$ npm install hexo-deployer-git –save

if you have the error:

ERROR Deployer not found: git
